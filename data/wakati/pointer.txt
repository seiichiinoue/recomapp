C ++ の ポインタ について 色々 と 調べ た ので メモ ． 
アドレス と ポインタ 
アドレス は ， 値 が 格納 さ れ て いる メモリ の 位置 の こと ． 変数 の アドレス は 以下 の よう に し て 取得 する こと が できる ． 
ポインタ 変数 ( ポインタ ) は アドレス を 保持 する 変数 の こと ． 以下 の よう に 宣言 を し て 使う こと が できる ． 
以下 の よう に 間接 参照 を 行う こと で ポインタ が 指し て いる 値 に アクセス する こと も できる ． 
配列 と ポインタ 
以下 の 通り ， 配列 は 通常 ， 配列 の 先頭 を 指す ポインタ へ と 変換 さ れる ． 
また ， が 配列 の 先頭 の ポインタ で ある こと から ， と は 配列 の 同じ 要素 を 示す ． 
つまり 以下 の 通り ， は 同じ アドレス に なっ て いる ． 
C 言語 の 作者 による 論文 : The Development of the C Language に よる と ， C 言語 の 祖先 で ある B 言語 や NB という 言語 の 都合 が ある らしい が ， そこら 辺 は 割愛 する ． 
追記 : しかし 厳密 に は ， を 行なっ た 際 の 挙動 が 違う こと など から ， 「 a は 配列 で あり ， ほとんど の 場合 配列 の 先頭 を 指す ポインタ へ と 変換 さ れる 」 という の が 正しい 表現 で ある と @ hsjoihs 先生 より ご 指摘 いただい た ． 
関数 の 引数 
ポインタ 渡し は 以下 の 通り ． 引数 を ポインタ 変数 と し ， 引数 に アドレス を 渡す こと で 渡さ れ た アドレス の 値 を 間接 参照 する こと で ， 関数 の 呼び出し 元 の 変数 を 書き換える こと が できる ． 
参照 渡し は 以下 の 通り ． 参照 渡し は C に は なく ， C ++ で 新た に 追加 さ れ た 記法 らしい ． 引数 で 与え られる 変数 の 別名 を つける 記法 と 解釈 できる ．
